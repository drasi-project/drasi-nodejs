# Security Policy

The Drasi maintainers take the security of our projects seriously, including
`@drasi/lib` and all source code managed through the
[drasi-project](https://github.com/drasi-project) GitHub organization.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, report them by email to [security@drasi.io](mailto:security@drasi.io).
You should receive a response within 24 hours. If for some reason you do not,
please follow up by email to ensure we received your original message.

To help us triage quickly, please include as much of the following as you can:

- Type of issue (e.g. memory safety/FFI, supply-chain, injection, etc.)
- Full paths of the source file(s) related to the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Preferred Languages

We prefer all communications to be in English.

This repository's policy mirrors the organization-wide
[drasi-project security policy](https://github.com/drasi-project/.github/blob/main/SECURITY.md).
