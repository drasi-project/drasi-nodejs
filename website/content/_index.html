@drasi/lib runs Drasi's continuous-query engine in-process in Node.js — no servers, no brokers, no Kubernetes. Define sources and reactions in plain JavaScript, or load native Drasi plugins at runtime.

Get started View on npm

npm install @drasi/lib

{{< /blocks/cover >}} {{< blocks/lead color="dark" >}}

Drasi is a change-driven data platform: you write continuous queries over your data, and Drasi keeps their results up to date as the underlying data changes — pushing you the added, updated, and removed rows.

@drasi/lib brings that engine directly into your Node.js process so you can react to change in real time from your own application code.

{{< /blocks/lead >}} {{% blocks/section color="light" type="row" %}} {{% blocks/feature icon="fas fa-plug" title="In-process, zero infrastructure" %}} A native ([napi-rs](https://napi.rs)) addon hosts the embeddable Drasi engine inside Node. Prebuilt binaries ship for Windows, Linux (x64/arm64) and Apple-silicon macOS — no Rust toolchain required. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-js" title="Sources & reactions in JavaScript" %}} Push graph changes from your own code with `pushChange`, and handle query result diffs with a plain JS callback via `addJsReaction` — no Rust, no plugins needed. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-cubes" title="Dynamic native plugins" %}} Discover and load Drasi source / reaction / bootstrap plugins at runtime, or pull them straight from the `ghcr.io/drasi-project` OCI registry — exactly like `drasi-server`. {{% /blocks/feature %}} {{% /blocks/section %}} {{% blocks/section color="white" type="row" %}} {{% blocks/feature icon="fas fa-code" title="First-class TypeScript" %}} Every config, result, and callback payload is exposed with a concrete type in the generated `index.d.ts`, plus a typed `DrasiErrorCode` enum for error handling. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-stream" title="Streaming & metrics" %}} Subscribe to component events and logs, and read per-query / per-reaction and lifecycle metrics from the running engine. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-book" title="Learn by example" %}} Start from the [Getting started](docs/getting-started/) guide, then explore the [end-to-end trading demo](docs/examples/trading/) that joins live prices against a Postgres database. {{% /blocks/feature %}} {{% /blocks/section %}}
